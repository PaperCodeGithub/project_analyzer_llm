# Project Analyzer

## Overview

**Project Analyzer** is an AI-powered repository intelligence tool. It crawls your entire codebase, processes every file into **high-dimensional vectors**, and stores them in **ChromaDB**. Using a **Large Language Model (LLM)**, it allows you to "talk" to your code, Identifying bugs, explaining complex logic, and generating new features based on the existing architecture which is the ultimate utility for any open-source developer trying to jump into a complex repository.

***This README.md file is actually generated by this model itself!***

## Usage
You can ask questions about the codebase. The system will use the Large language model to understand your question and retrieve relevant code snippets to answer it.

Example Questions:

*   "How do I implement authentication?"
*   "What is the purpose of the `calculate_something` function?"
*   "How do I use the database connection?"
*   "Write `README.md` for this project."
*   "Explain the entire workflow."

## File Types Supported

The system supports indexing these file formats:
- Source Code: `.py`, `.js`, `.ts`, `.c`, `.cpp`, `.rs`, `.h`
- Markup/Text: `.html`, `.txt`, `.md`

## Technology Stack

* **Python 3.10** Programming language
*  **LangChain**
   - Used to build chains that integrate document retrieval with language models.
   - Facilitates building applications that use LLMs over custom knowledge bases.
* **ChromaDB**
   - A vector database used to store and query embeddings of code chunks.
   - Enables semantic search over the indexed codebase.
* **Qwen3-Coder / ChatOllama**
   - The language model used for understanding queries and generating responses is `qwen3-coder:480b-cloud`.
   - Accessed via `ChatOllama`, which interfaces with Ollama-hosted models like Gemma3.

* **HuggingFace Embeddings**
   - Specifically using `sentence-transformers/all-MiniLM-L6-v2` for creating vector representations of text.
   - Integrated through `HuggingFaceEmbeddings`.

* **Colorama**
   - For colored terminal output during logging/indexing steps (e.g., `Fore.LIGHTGREEN_EX`).
* **Rich**
  - For formatting markdown output of AI in terminal.

## Workflow

1.  The user provides a path to the codebase.
2.  The `scan_and_load_codebase` function indexes the codebase, splitting it into chunks and storing the embeddings in Chroma.
3.  The user asks a question about the codebase.
4.  The `create_retrieval_chain` retrieves relevant documents from Chroma based on the question.
5.  The retrieved documents are fed to the language model (ChatOllama) via the `ChatPromptTemplate`.
6.  The language model generates an answer, which is then printed to the console.

## Setup & Usage

### 1. Clone and Install Dependencies

Clone this repository and navigate into the project folder:

```bash
git clone <repository-url>
cd project_analyzer_llm
pip install -r requirements.txt
```

Ensure all necessary libraries such as `langchain`, `chromadb`, `sentence-transformers`, and optionally `colorama` are installed.

## Customization Options
You can customize various aspects of the system:

#### Change Ollama model
Modify the model passed to `scan_and_load_codebase()`:

```python
scan_and_load_codebase(model="your_ollama_model")
```

#### Adjust Chunk Sizes

Tune chunk sizes in `RecursiveCharacterTextSplitter` calls within `splitters` dictionary:

```python
".py": RecursiveCharacterTextSplitter.from_language(
    language=Language.PYTHON,
    chunk_size=3000,
    chunk_overlap=300
)
```
Larger chunks provide broader context; smaller ones increase precision.

#### Add More File Types
Update the `suffixes` list in `scan_and_load_codebase()` to support additional languages:

```python
suffixes = ['.py', '.js', '.ts', '.java', '.go']  # Add Java and Go support
```

Also update `splitters` accordingly with appropriate language parsers.
